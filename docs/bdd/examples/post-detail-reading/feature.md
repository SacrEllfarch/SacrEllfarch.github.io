# Feature: 文章详情页阅读体验

## 来源

- PRD：来自用户对文章详情页的阅读体验要求。
- SDD：可对应 `docs/sdd/post-detail-reading/`，若该目录尚未创建，则以现有实现和对话记录为需求来源。
- 关联页面：`/posts/ask_first`

## 用户故事

作为博客读者，我希望阅读文章时能够看到当前位置、通过目录平滑跳转，并看到顶部阅读进度，以便更容易掌握长文结构和阅读进度。

## 行为范围

- 右侧目录根据读者阅读位置高亮对应标题。
- 点击目录项时页面平滑滚动到对应段落。
- 文章顶部显示阅读进度条，并随滚动更新。
- 文章标题使用与目录一致的竖向指示线视觉语言。

## 非范围

- 不验证文章内容本身的文案正确性。
- 不验证第三方访问统计服务的准确计数，只验证页面能展示接口返回值或占位状态。
- 不引入自动化 BDD runner。

## 业务规则

- 目录项不能使用普通同页 hash 链接触发默认瞬移。
- 阅读进度条必须在文章详情页可见，并不能引发 SSR 水合白屏。
- 目录和标题不能出现横向滚动条。

## 验收摘要

- [ ] 打开文章详情页后目录正常渲染。
- [ ] 滚动到不同标题附近时目录 active 项同步变化。
- [ ] 点击目录项时页面以动画滚动到目标段落。
- [ ] 顶部阅读进度条随页面滚动增加。
- [ ] 刷新文章详情页不白屏。
